# ZHERP SVN 审查日志模板

以下模板以既有样例文件为基准。生成审查日志时保持栏目顺序和表达口径一致。

```markdown
# ZHERP 当日 SVN 提交审查日志

日期：<YYYY-MM-DD>

审查范围：<时间窗说明>内共 <N> 个 revision，分别为 <revision 列表>。

总体结论：<一句话概括问题数量、最高风险级别和建议动作。>

## 提交概览

| Revision | 提交人 | 提交时间 | 提交说明 | 结论 |
| --- | --- | --- | --- | --- |
| <revision> | <author> | <commit_time> | <message> | <未发现明显问题 / 有高风险问题 / 有建议优化 / 缺少 diff，结论受限> |

## 变更概览

- <revision>：<概括该 revision 的主要变更。>
- <revision A> / <revision B>：<可合并概括同类变更。>

## 关键风险

- <关键风险 1>
- <关键风险 2>
- <如果无关键风险，写“未发现需要单独提示的关键风险。”>

## 问题清单

<只输出实际发现问题的级别。没有问题的 P 级别不要写，不要写“未发现”。如果 P0/P1/P2/P3 全部没有问题，本节只写：>

未发现需要列入问题清单的问题。

<如果存在问题，按实际严重级别输出下面格式；未出现的级别整体省略。>

### P<级别>

#### <问题标题>

- 相关 revision：<revision>
- 相关文件：
  - [<文件名>](<绝对路径或可点击路径>#L<行号>)
- 理由：<为什么这是问题。>
- 影响：<会造成什么后果。>
- 修改建议：<建议怎么改。>

## 验证

- <SVN 日志、diff、update、Maven 构建、实体生成等验证情况。>
- <如果部分 revision 缺少 diff，必须写“部分 revision 缺少 diff，结论受限”。>
- <说明本次审查只基于成功取得的日志和 diff，不依赖猜测性补全。>

## 总体审查结论

<用一段话给出最终结论：整体是否可接受，哪些 revision 必须先修，哪些只是建议优化，是否存在受限结论。>
```

无新增提交时使用简短审查日志：

```markdown
# ZHERP 当日 SVN 提交审查日志

日期：<YYYY-MM-DD>

审查范围：<时间窗说明>内无新增提交。

总体结论：无新增提交。

## 验证

- 已完成 SVN 最小认证验证。
- 已按指定时间范围拉取 SVN 日志。

## 总体审查结论

本次审查范围内无新增提交。
```

日志全被跳过模式过滤时使用：

```markdown
# ZHERP 当日 SVN 提交审查日志

日期：<YYYY-MM-DD>

审查范围：<时间窗说明>内存在提交，但全部命中跳过模式，未进入代码审查。

总体结论：无需要审查的提交。

## 验证

- 已完成 SVN 最小认证验证。
- 已按指定时间范围拉取 SVN 日志。
- 已按跳过规则过滤提交说明。

## 总体审查结论

本次审查范围内无需要审查的提交。
```
